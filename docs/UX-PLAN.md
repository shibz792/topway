# UX Plan

## Primary Journey
Employer lands -> understands Sri Lankan workforce offer -> sees licence and operational proof -> reviews categories/process/markets -> selects Request Workers -> completes progressive requirement form.

## Secondary Journey
Candidate uses the secondary navigation link -> reviews safety and process guidance -> registers interest separately.

## Homepage
1. Hero and Request Workers
2. Compact trust bar
3. Three recruitment categories
4. Four employer outcomes
5. Five-stage process
6. Four selected markets
7. Licence proof
8. Final employer CTA

Remove the industry catalogue, dual-audience introduction and full enquiry form from the homepage.

## Navigation
Desktop: Services, Employers, Global Markets, About, Licence, Contact, For Candidates, Request Workers.
Mobile: Request Workers first, six primary links, candidate link, Call and WhatsApp.

## Content Rules
- Employer action is visually dominant.
- One primary CTA per section.
- Supporting pages hold detail; homepage provides decision support.
- Never imply active markets, vacancies, guarantees or unsupported outcomes.
